# Accessibility (Phase 8.4)

Aegis targets **WCAG 2.1 AA** on every route in `docs/routing.md`. This
document is the checklist a reviewer walks before shipping any change
that touches interactive UI, animation, or new copy.

## Automated coverage

| Tool | Scope | How to run | Owner |
| --- | --- | --- | --- |
| `axe-core` via Playwright | Public routes (`/`, `/auth`, `/auth/reset-password`) | `bun run test:a11y` | CI |
| i18n id coverage | Every `i18n._()` / `t()` / `<Trans>` call site | `bun run test` (included in default vitest run) | CI |
| Reduced-motion CSS + `useReducedMotion()` guards | Framer Motion + Tailwind animations | Manual toggle in DevTools → Rendering → Emulate CSS media | Reviewer |

`bun run test:a11y` boots the running dev server on `http://localhost:8080`
and expects zero **critical** or **serious** violations. `color-contrast`
and `region` are intentionally excluded — see the note at the top of
`tests/e2e/a11y-axe.spec.ts` for the reasoning.

Authenticated / locked routes are covered by
`tests/e2e/locale-switch.spec.ts`, which is auto-skipped without an
injected Supabase session.

## Keyboard-only flows

Every flow below must be completable **without touching the mouse** on
both light and dark themes. Reviewer tabs from the browser address bar
into the page and works forward with `Tab` / `Shift+Tab` / `Enter` /
`Space` / arrow keys only. Focus rings must be visible on every stop.

| Flow | Route entry | Expected keyboard path |
| --- | --- | --- |
| Onboarding | `/onboarding` | Tab → passphrase → confirm → continue; recovery-code screen: Tab to "I've saved this" → Enter |
| Unlock | `/lock` | Passphrase field is auto-focused; Enter submits; focus returns to field on error |
| Add (manual) | `/vault/new` → Manual tab | Tab through Issuer / Account / Secret / Advanced → Save |
| Add (scan) | `/vault/new` → Scan tab | Space toggles camera permission prompt; "Add by hand" link is Tab-reachable |
| Import | `/vault/import` | File picker opens on Enter; conflict resolver rows are Tab-reachable, Space activates the radio |
| Export | Profile → Vault → Export | Passphrase field auto-focus, Enter downloads |
| Change passphrase | Security → Change passphrase | Old → New → Confirm → Save, all Tab-reachable |
| Delete account | Profile → Danger zone → Delete | Typing the confirmation string enables the button; Enter submits |
| Locale switch | Profile → Language | Row opens sheet; arrow-key or Tab through options; Enter selects; sheet closes and focus returns to the trigger row |

If any of the above requires the mouse to complete, file it under
"a11y bug" and block the release.

## Reduced motion

- `src/styles.css` includes a `@media (prefers-reduced-motion: reduce)`
  block that neutralises every CSS animation + transition site-wide
  (safety net).
- Framer Motion consumers that render essential motion (scanner sweep,
  onboarding slides, account-card flip, chrome tap feedback) already
  call `useReducedMotion()` and short-circuit the animation. New motion
  components must do the same — reach for `useReducedMotion()` before
  writing `transition` props.

## Colour + contrast

Reviewed manually in `/dev/tokens` (renders every token in Auto / Light /
Dark). Any new semantic token must resolve to a value that clears **4.5:1**
against its intended surface in both themes. Tokens that fall below —
`PLACEHOLDER`, disabled states — are explicitly for non-informational
text and never used for content the user must read.

## What we intentionally do not automate

- Full keyboard walk of authenticated flows. Playwright can drive them,
  but the failure modes (visible focus ring, focus-return-on-close) are
  visual and better caught in review than encoded as brittle selectors.
- Screen reader labels for framer-motion transitions. Reviewer runs
  VoiceOver / TalkBack once per release against the flows above.
- Screenshot regression across every route. The harness lives at
  `tests/e2e/locale-switch.spec.ts` (Playwright) and can be extended
  route by route — we add snapshots as flows stabilise, not up-front.
